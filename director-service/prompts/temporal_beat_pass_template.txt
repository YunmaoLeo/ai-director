## Task: Global Beat Planning (Pass 1 of 3)

Divide the timeline into narrative beats. Each beat is a coherent story segment with a goal and mood.

Output a JSON object with:
- beats: array of beat objects

Each beat must have:
- beat_id: unique identifier (e.g., "beat_1")
- time_start: start time in seconds
- time_end: end time in seconds
- goal: what this beat achieves narratively
- mood: emotional tone (e.g., "calm", "tense", "dramatic", "contemplative")
- subjects: array of object IDs that are focus subjects during this beat

Guidelines:
- Beats must cover the full time span [{time_start}s, {time_end}s]
- Beats should not overlap
- 2-5 beats is typical for a {duration}s timeline
- Align beat transitions with events when possible
- Consider subject visibility windows
- Build emotional rhythm, not equal slices only:
  - Setup / orientation beat
  - Development / pursuit beat(s)
  - Peak event beat
  - Resolution beat
- If events are dense, group them into coherent dramatic phases rather than one beat per event.

Reference beat style example (do not copy literally):
- beat_1: orient space and actors, establish stakes
- beat_2: rising interaction or pursuit, increase urgency
- beat_3: decisive event window, maximum emphasis
- beat_4: aftermath or resolution, release tension

Output ONLY valid JSON.
